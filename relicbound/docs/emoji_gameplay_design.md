## 二、表情包在像素公司里的玩法定位

表情包系统不是装饰，是员工之间的**轻量社交货币**。它应该：

1. **即时反馈**：聊天中发一个表情，其他人立刻看到，形成"我说-你回"的节奏
2. **场景关联**：不同表情对应办公室不同物件（咖啡机、鱼竿、柯基等），让办公室世界活起来
3. **情绪表达**：点赞表示认可，吃瓜表示围观，其他表情传递不同态度

当前问题：
- 点赞图标不像点赞（已修）
- 表情包只有静态展示，没有交互反馈（点击、动画、连锁反应）
- 表情和公司资产（咖啡机、鱼、柯基）没有联动

## 三、优先级排序

### P0 - 必须完成（Milestone 2 内）
1. **点赞表情修复**：SVG 路径已改，确认显示正常。（阿默已完成，待验收）
2. **表情点击反馈**：点击表情后要有视觉反馈（放大弹跳、颜色闪烁等），让玩家感知到"我发了一个东西"
3. **表情与公司资产联动**：
   - 点赞 -> 鱼缸里的锦鲤摇尾巴
   - 咖啡表情 -> 咖啡机冒蒸汽
   - 柯基相关表情 -> 柯基起身或转头

### P1 - 应该做（Milestone 3 内）
4. **表情连击系统**：连续发同一种表情触发特殊效果（比如连续5个点赞，办公室全员鼓掌）
5. **表情收藏/快捷栏**：玩家常用表情放在快捷栏，一键发送
6. **表情作为成就触发器**：集齐所有办公室表情解锁隐藏对话

### P2 - 可以做（后续迭代）
7. **表情故事线**：特定表情组合触发剧情事件
8. **表情交易市场**：员工之间交换稀有表情

## 四、验收标准

每个表情包功能必须通过以下测试：

### 视觉验收
- [ ] 点赞图标在 16x16、32x32、64x64 三个尺寸下都清晰可辨
- [ ] 所有表情在深色/浅色聊天背景上都有足够对比度
- [ ] 表情 SVG 路径无断裂、无重叠、无溢出

### 交互验收
- [ ] 点击表情有明确的视觉反馈（至少 200ms 的缩放动画）
- [ ] 表情出现在聊天框后，新消息滚动时表情跟随移动不丢失
- [ ] 浏览器插件端（Amazon extension）和公司主站端表情一致

### 玩法验收
- [ ] 点赞表情能触发至少一种办公室联动效果
- [ ] 表情发送延迟不超过 500ms（本地测试）
- [ ] 表情系统不影响聊天消息的正常收发

## 六、设计原则备忘

- 表情包是社交工具，不是摆设。每个表情都要有"用了它，世界会有变化"的感觉。
- 不要为了加表情而加表情。每个新表情都应该服务于办公室世界的沉浸感。
- 老板关注细节，所以像素级的视觉质量不能妥协。
- 先做最小可玩版本，再迭代。不要一次性设计 20 个表情。
