## 表情包系统设计决策 (2026-06-30)
- 表情包是社交货币，不是装饰。每个表情必须有交互反馈和世界联动。
- 点赞表情修复：SVG 路径从两条分离矩形改为手掌+竖拇指像素风（core.js + content.js）。
- 优先级：P0=点击反馈+资产联动（鱼/咖啡机/柯基）；P1=连击系统+快捷栏；P2=表情故事线+交易。
- 验收标准：三个尺寸清晰度、点击缩放动画>=200ms、联动延迟<=500ms、插件端与主站一致。
- 设计原则：最小可玩版本优先，每个表情服务于办公室沉浸感，像素级视觉质量不可妥协。
- 相关文件：relicbound/docs/emoji_gameplay_design.md
