# 聊天到像素宇宙沉淀流水线

## 目标

Hermes Pixel Works 的聊天记录不再只是一次性气泡。老板的方向、员工的灵感、团队的吐槽和承诺，会被沉淀为像素宇宙的一部分，并逐步变成场景、资产、小游戏、角色对白和可执行任务。

## 沉淀层级

1. 原始聊天
   - 保留在 Kanban 任务和日志中。
   - 不直接塞入长期记忆，避免上下文膨胀。

2. 宇宙事件
   - 写入 `company_state.json` 的 `universe_events`。
   - 记录“谁说了什么，这句话属于公司生活、玩法、地图、叙事还是系统功能”。

3. 宇宙灵感
   - 写入 `universe_ideas`。
   - 例如“柯基追鱼”“奶茶外卖”“锦鲤保佑”“街区入口”。

4. 宇宙任务候选
   - 写入 `universe_tasks`。
   - 明确可落地的聊天内容会创建 `[宇宙沉淀]` 看板任务。

5. 真实落地
   - 小韩定义玩法和验收。
   - 阿默改真实文件。
   - 小研检查冲突和可玩性。
   - 小文把聊天梗转成对白、道具说明或剧情事件。

## 会被沉淀的内容

- 老板明确提出的方向、功能、项目、世界设定。
- 员工关于公司生活的可复用梗：柯基、鱼缸、奶茶、咖啡机、外卖、水吧等。
- 玩法和小游戏想法：平台跳跃、ARPG、Roguelike、关卡、战斗、手感等。
- 地图和空间想法：公司外、街区、洗手间、员工家、别墅、入口等。
- 角色、对白、剧情、世界观相关内容。
- 员工承诺“我来做”“我负责”“先做一版”等。

## 不会自动变成任务的内容

- 单纯寒暄。
- 无明确对象的短吐槽。
- 执行器错误、Traceback、系统异常消息。
- 机械式回复或无意义的“收到”。
