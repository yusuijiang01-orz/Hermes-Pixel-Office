# 主动巡检与资料研究机制 PRD

生成时间：2026-06-30

适用对象：
- 老板
- 小韩（策划主编）
- 阿默（主程 / 制作人）
- 小研（玩法研究 / 测试）
- 小文（叙事 / 文案 / UI）

## 1. 问题定义

Hermes 当前最大的组织问题，不是“没有员工”，而是员工默认处于被动响应状态：

- 老板不催，任务不动
- 老板不点名，员工不主动查资料
- 群里会聊很多，但没有稳定沉淀为世界变化
- 员工即使知道要做游戏，也常常停在补 bug、回消息、解释阻塞
- 资料研究没有被当成正式工作流，而像是临时附加动作

这会导致一个坏循环：

1. 老板发需求
2. 员工讨论
3. 讨论被 bug、执行器、临时回复打断
4. 没有主动巡检和资料研究补位
5. 结果又回到“催一下动一下”

这份 PRD 的目标，是把 Hermes 团队从“被动聊天团队”升级为“主动巡检、主动研究、主动沉淀、主动汇报”的宇宙生产团队。

## 2. 目标

### 2.1 主目标

建立一套固定机制，让 4 个员工即使没有被老板逐条催促，也会：

- 主动巡检当前主线
- 主动查找参考资料
- 主动把聊天内容转成世界任务
- 主动输出阶段性交付
- 主动把结果回流到世界看板和宇宙沉淀

### 2.2 成功标准

如果机制生效，应出现以下现象：

- 老板 30-60 分钟内总能看到可见交付
- 员工不再只在被点名后才动
- 世界看板上能看到资料研究记录
- 聊天内容会持续长成任务、入口、物件、剧情和小游戏内容
- 员工回复“已完成”时，能明确指出它落在宇宙里的哪里

## 3. 核心原则

### 3.1 聊天只是燃料

聊天本身不是交付。聊天的作用是为宇宙提供：

- 方向
- 梗
- 决策
- 角色关系
- 世界设定
- 玩法灵感

如果聊天最后没有转成任务或世界内容，就只是噪音。

### 3.2 资料研究是正式工作，不是附加动作

资料研究不再被视为“有空再查”，而必须成为员工的常规职责。

它的作用不是抄袭，而是：

- 找同类作品的成熟解法
- 判断哪些风格适合 Hermes
- 找到该避免的坑
- 为当前版本提供更稳的落地参考

### 3.3 主动巡检优先于被动等待

员工不能默认“等老板下一句话”。

只要主线项目还在推进，员工就应该主动巡检：

- 现在卡在哪
- 缺什么资料
- 缺什么关卡设计
- 谁应该接下一棒
- 哪些聊天内容应该沉淀进世界

### 3.4 研究必须回流宇宙

查资料不是为了写一段漂亮总结，而必须回流为至少一种结果：

- 新任务
- 新参数建议
- 新关卡结构
- 新视觉方向
- 新角色文案
- 新小游戏入口
- 新世界事件

## 4. 角色机制

## 4.1 小韩：主动策划巡检

职责定位：
- 主线版本经理
- 玩法闭环负责人
- 任务收敛者

固定巡检动作：
- 每 30-60 分钟巡检当前主线版本一次
- 检查是否已有基础关卡设计
- 检查是否已有明确验收标准
- 检查是否有任务漂移、目标发散、多人重复劳动

主动研究职责：
- 查找同类玩法结构
- 提取关卡结构模板
- 拆出最小可玩闭环

必须产出：
- 版本目标
- 任务拆分
- 基础关卡提纲
- 阶段验收标准

禁止行为：
- 群里聊得很多，但不给明确收敛
- 让员工同时做太多平行方向

## 4.2 阿默：主动交付巡检

职责定位：
- 真实文件落地负责人
- 宇宙入口接入负责人
- 独立小游戏与母世界连接负责人

固定巡检动作：
- 每 30-60 分钟检查一次当前主线是否已有可演示入口
- 检查小游戏是否已正确挂接回宇宙入口
- 检查返回路径是否稳定
- 检查移动端是否能玩

主动研究职责：
- 查找合适的技术落地方式
- 查找类似项目的页面结构、移动端交互、入口组织方式
- 比较“独立开发”和“宇宙接入”的最佳实践

必须产出：
- 可打开入口
- 文件路径
- 当前版本可玩的内容
- 已知阻塞与修复建议

禁止行为：
- 没有关卡设计就直接开做
- 只修 bug，不给可见交付
- 做出一个宇宙里没有入口的独立页面

## 4.3 小研：主动资料研究巡检

职责定位：
- 参考资料负责人
- 玩法风险负责人
- 可玩性验证负责人

固定巡检动作：
- 每 30-60 分钟至少执行一次研究或测试巡检
- 检查当前主线还缺什么外部参考
- 检查当前方案有哪些手感或结构风险
- 检查是否需要补参数建议和测试清单

主动研究职责：
- 查找同风格小游戏
- 查找像素平台跳跃 / 闪避 / 配送类案例
- 提炼可借鉴机制
- 明确不适合 Hermes 的部分

必须产出：
- 参考对象
- 借鉴点
- 风险点
- 参数建议
- 测试清单

禁止行为：
- 只说“我去查”
- 只否定方案，不给可执行建议
- 查了资料却不回流到任务或看板

## 4.4 小文：主动内容沉淀巡检

职责定位：
- 世界观内容负责人
- 聊天梗转内容负责人
- 入口文案与结算文案负责人

固定巡检动作：
- 每 30-60 分钟检查一次最近聊天是否有可沉淀内容
- 检查当前主线是否缺入口说明、角色对白、提示文本、结算文案
- 检查小游戏完成后是否有回流母世界的纪念内容

主动研究职责：
- 查找类似像素游戏的文案风格
- 查找轻喜剧像素世界的对白组织方式
- 查找适合 Hermes 的 UI 文案与提示风格

必须产出：
- 世界动机文案
- 入口文案
- 提示文案
- 结算文案
- 聊天梗沉淀记录

禁止行为：
- 只写气氛词
- 没有具体文本可以直接接进游戏

## 5. 固定节奏机制

## 5.1 巡检周期

所有主线项目默认启用：

- 巡检周期：每 30 到 60 分钟
- 最长无交付时长：60 分钟
- 超时后状态：标记为“推进失效”

## 5.2 巡检结果类型

每次巡检必须落成以下之一：

- 新任务
- 新研究记录
- 新关卡设计
- 新入口接入
- 新文本沉淀
- 新测试结果
- 新阻塞说明
- 新验收汇报

## 5.3 空转识别

系统应识别以下空转行为：

- 反复“收到”
- 反复“我在处理”
- 讨论很长，但没有新增任务或世界变化
- 长时间只在补非阻塞 bug

一旦识别为空转，应在看板中显示：

- 空转时长
- 当前责任人
- 最近一次真实交付时间
- 建议下一步

## 6. 资料研究工作流

## 6.1 研究触发器

以下情况必须触发研究：

- 新小游戏立项
- 新关卡设计开始前
- 当前方案明显缺参考
- 手感或结构不确定
- 员工提出“像某种风格”的目标

## 6.2 研究记录模板

每条研究记录至少包含：

- 研究主题
- 研究人
- 研究时间
- 参考对象
- 借鉴点
- 不适合 Hermes 的点
- 准备落地到哪个项目
- 建议下一步

推荐结构：

```md
主题：Mushroom Runner 基础关卡参考
研究人：小研
参考对象：某平台跳跃小游戏 A / B / C
借鉴点：
- 开场 10 秒内给到一次低门槛成功反馈
- 第一关只教 1 个核心动作
不适合 Hermes：
- UI 太拥挤
- 世界观过于黑暗，不适合 Hermes 轻喜剧像素气质
落地建议：
- 用街机屏入口
- 第一关只教移动与跳跃
```

## 6.3 研究不是收藏夹

研究记录必须指向真实任务，否则不算有效研究。

有效研究：
- 已生成任务
- 已附着到某个游戏
- 已变成参数建议或关卡结构

无效研究：
- 只贴链接
- 只说“这个不错”
- 没说明如何服务当前项目

## 7. 聊天到宇宙内容的回流机制

## 7.1 聊天分类

聊天内容应至少被分流到以下池子：

- `task_queue`
  明确可执行项
- `research_queue`
  需要查资料的方向
- `world_ideas`
  地图、区域、入口、物件
- `gameplay_ideas`
  玩法、机制、关卡
- `story_bits`
  对白、关系、世界事件
- `blocking_items`
  需要老板拍板的问题

## 7.2 回流要求

每条高价值聊天在 30-60 分钟内，至少应产生一种回流：

- 看板新增一条卡片
- 宇宙沉淀新增一条事件
- 某个员工新增一条研究记录
- 某个小游戏新增一个入口或内容点
- 某个员工私聊草稿被自动准备好

## 7.3 回流优先级

最高优先：
- 老板明确要求
- 当前主线相关方向
- 可直接变成关卡或入口的内容

中优先：
- 可作为剧情、对白、物件的内容

低优先：
- 纯日常闲聊

## 8. 看板联动要求

这套机制必须能在世界看板中被老板直接看到。

老板需要直接看到：

- 过去 30-60 分钟谁交付了什么
- 谁查了什么资料
- 哪些聊天已经沉淀为宇宙内容
- 当前主线卡在哪
- 哪个员工该被私聊推动

换句话说，这套机制如果只存在于后台日志里，就算失败。

## 9. 自动化建议

系统后续应支持：

- 定时巡检当前主线任务
- 自动生成研究提醒
- 自动提示某员工“该交一次可见成果了”
- 自动把高价值聊天放入研究队列
- 自动把研究结果挂到对应游戏任务

## 10. 验收标准

这套机制落地后，至少满足：

1. 员工不再只在老板催促后才行动
2. 世界看板能看到研究记录
3. 每 30-60 分钟能看到一次可见交付
4. 高价值聊天能持续转成任务、入口、物件、剧情或关卡
5. 小研能稳定承担资料研究职责，而不是临时被点名
6. Hermes 宇宙开始表现出“自我生长”的特征

## 11. 一句话结论

Hermes 不应再依赖“老板催一下，员工动一下”的被动循环，而必须形成：

`主动巡检 -> 主动研究 -> 主动沉淀 -> 主动汇报 -> 世界持续生长`
