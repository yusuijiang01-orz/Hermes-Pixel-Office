# Hermes Pixel Office 恢复清单

## 恢复原则

1. 先恢复主世界办公室稳定性，再恢复编辑模式，再恢复移动端交互，最后恢复宇宙扩展与 Agent 感知。
2. 每恢复一个功能都必须做 PC 与手机两端验证。
3. 不再用“看起来差不多”的 UI 替代原来的交互逻辑。
4. 不再覆盖用户已经修好的功能；修改前必须备份当前文件。
5. 所有场景、物品、角色坐标应从统一数据源读取，避免“聊天说加了微波炉但世界里看不到”。

## P0 立即恢复：主世界不能坏

- [ ] 主世界办公室恢复为完整布局，不允许出现空办公室。
- [ ] 普通模式必须显示：办公桌、显示器、键盘、鼠标、电脑主机、鱼缸、水吧、咖啡机、制冰机、打奶泡、茶几、绿植、员工、柯基等已有物体。
- [ ] 普通模式与编辑模式使用同一套场景物品数据，避免两边看到的物体不一致。
- [ ] 清理或兼容损坏的本地坐标缓存，避免旧 `localStorage` 把物体挤到屏幕外。
- [ ] 页面加载后员工直接出现在合理状态或工位附近，不再从左上角缓慢走到工位。
- [ ] 世界时间与“长连接实时”秒数正常跳动。
- [ ] 公司夜晚有人时应开灯，只有办公室无人时才允许黑灯状态。

验收：
- 刷新 3 次，办公室布局稳定。
- 手机和 PC 都能看到同一批核心物体。
- 员工不再从左上角集体走回工位。

## P0 立即恢复：编辑模式最终版

- [ ] 底部保留清晰的 `CHAT 聊天` 与 `编辑` 两个入口，互不冲突。
- [ ] 顶部菜单恢复为目标版本：`编辑 / 设置 / 视角 / 关闭编辑`。
- [ ] 点击 `关闭编辑` 只退出编辑模式，不关闭整个办公室 UI。
- [ ] 进入编辑模式时，必须基于普通办公室当前布局，不允许进入一个缺物体的编辑场景。
- [ ] 所有可编辑物体都能被选中：办公桌、显示器、键盘、鼠标、电脑主机、鱼缸、水吧、咖啡机、制冰机、打奶泡、茶几、绿植、书柜、微波炉等。
- [ ] 点击物体后，物体边缘出现白色虚线，表示选中。
- [ ] 手指或鼠标按住拖拽时，物体必须实时跟随移动，不能等保存后才跳到新位置。
- [ ] 放开后物体停留在当前位置，再次拖拽可继续移动。
- [ ] 物体重叠或进入不可放置区域时，虚线变红。
- [ ] 位置非法时底部只显示 `×`，隐藏 `√`。
- [ ] 位置合法时底部显示 `√` 和 `×`。
- [ ] 点击 `√` 保存该物体新坐标，并立即渲染，不需要浏览器刷新。
- [ ] 点击 `×` 取消本次移动，物体回到移动前坐标。
- [ ] 点击保存或关闭编辑模式不能卡死。
- [ ] 保存后的坐标刷新页面后仍然存在。
- [ ] 场景物品坐标拆到独立文件或独立 API，不再硬塞进主页面逻辑。
- [ ] 新增物体要有明确入口：类型、名称、尺寸、坐标、碰撞盒、可编辑开关、是否动画。

验收：
- 在手机端移动一张办公桌，实时跟手，保存后刷新仍在新位置。
- 故意把桌子拖到鱼缸上，虚线变红且 `√` 消失。
- 点 `×` 后物体回到原位。
- 编辑模式和聊天模式按钮不再共用。

## P0 立即恢复：手机操作

- [ ] 手机端单指拖动画布，可查看看不见的范围。
- [ ] 手机端双指缩放世界，支持最小比例和最大比例限制。
- [ ] 编辑模式也复用单指拖动画布、双指缩放逻辑。
- [ ] 若正在拖拽物体，单指优先移动物体；若未选中物体，单指移动视角。
- [ ] 摇杆改成悬浮半透明大摇杆，尺寸足够手指操作。
- [ ] 右下角四个按钮扩大热区，手机上能稳定按到。
- [ ] 三个内置小游戏支持同一套移动端摇杆和按钮，不只支持键盘。
- [ ] 手机端 UI 不遮挡顶部模式切换，不遮挡聊天/编辑入口。

验收：
- iPhone Safari 中单指拖动画面流畅。
- 双指缩放有效，且不会无限缩小或放大。
- 摇杆和四个按钮单手可操作。

## P0 立即恢复：聊天实时性

- [ ] 手机版聊天列表打开时默认定位到底部最新消息。
- [ ] 群聊和私聊消息按真实发送时间排序。
- [ ] 不再浏览聊天记录时自动刷新跳动。
- [ ] 新消息到达时，如果用户已经在底部，则自动滚到底部；如果用户正在翻旧消息，不强制打断。
- [ ] 移动端聊天内容实时刷新，不能停留在昨天旧消息。
- [ ] 输入框文字颜色与背景必须有明显对比。
- [ ] 表情包、图片、文件按钮保持像素风格，不出现不统一黑框。
- [ ] 表情包可先选入输入区，再和文字一起发送；点击表情包不应立刻强制发送。
- [ ] 添加表情包后表情面板自动关闭。
- [ ] 浏览器插件端同步支持表情包、图片、收藏等聊天能力。

验收：
- 手机打开群聊直接看到最新消息。
- 发送一条消息后 PC 与手机 1 秒内看到。
- 翻旧消息时不会被自动拉回底部，除非用户回到底部。

## P1 恢复：全局返回与场景导航

- [ ] 所有房间、厕所、走廊、老板办公室、小游戏都必须有独立于场景逻辑之外的救命返回按钮。
- [ ] 主世界办公室、公司大厅、宇宙广场、小游戏之间建立统一场景栈。
- [ ] `返回` 回到上一个场景。
- [ ] `Home / 公司首页` 回到像素公司主界面。
- [ ] `Esc / Backspace` 在 PC 端也能触发安全返回。
- [ ] 进入厕所、小游戏、走廊、老板办公室后，不能出现出不来的情况。
- [ ] 点击员工进入聊天后，右上角返回应回到公司 UI 主界面，而不是再次返回聊天窗口。

验收：
- 从办公室进入厕所后，能一键回办公室。
- 从小游戏进入后，能回宇宙广场，再回公司大厅，再回主界面。

## P1 恢复：柯基状态机

- [ ] 使用用户提供的 160x160 单元精灵图。
- [ ] 支持状态：`idle / walk / run / bark / sleep`。
- [ ] 后续预留：`jump / sniff / happy`。
- [ ] 柯基比例要合理，手机端不能比员工大好几倍。
- [ ] 柯基不能缺腿、不能三条腿、不能倒着走。
- [ ] 移动方向与精灵朝向一致。
- [ ] 动画帧率与状态匹配：walk 慢，run 快，idle 轻微，sleep 慢。
- [ ] 柯基在办公室与游戏模式中使用同一套动画机，不出现旧狗/新狗混用。
- [ ] 柯基要有碰撞与寻路，不能穿墙、穿桌子、踩办公桌过去。
- [ ] 柯基可以跟随小文，但不能瞬移；需要路径、速度和状态过渡。
- [ ] 柯基可以自主行为：待机、走动、睡觉、靠近小文、靠近鱼缸、叫、玩耍。

验收：
- 手机和 PC 都显示同一只新版动画柯基。
- 柯基从 A 点到 B 点连续移动，不瞬移，不穿桌。
- 柯基走路时腿部动画自然，不是左右滑动。

## P1 恢复：办公室环境动画

- [ ] 鱼缸中的鱼继续游动，不能倒着游，不能游一会消失。
- [ ] 水草留在鱼缸内，手机端不跑出鱼缸。
- [ ] 水吧、咖啡机、制冰机、打奶泡恢复动画或状态反馈。
- [ ] 书柜等静态物体不能无故移动。
- [ ] 室外天气、日夜变化继续保留，但雨的方向要自然。
- [ ] 办公室有基础光影、阴影和灯光层次。
- [ ] 员工和老板角色有眼睛、表情，不再像僵硬无神的方块脸。
- [ ] 员工角色支持基础动态表情。

验收：
- 鱼缸动画 60 秒内连续正常。
- 夜间有人时办公室不是黑灯瞎火。
- 员工角色有眼睛和基础表情变化。

## P1 恢复：场景物品与环境感知

- [ ] 建立世界物品注册表，每个物品包含：`id / name / type / room / x / y / width / height / collision / interactive / owner / state`。
- [ ] 员工可查询自己所在房间有哪些物品。
- [ ] 员工可查询某个物品坐标，比如办公桌、厕所、水瓶、微波炉、柯基、鱼缸。
- [ ] 员工行动不再写死路线，而是基于物品坐标和可达路径。
- [ ] 小文知道柯基在哪里，可以走过去摸狗头。
- [ ] 员工知道自己的办公桌在哪里，桌子移动后员工能重新找到工位。
- [ ] 员工知道厕所、水吧、咖啡机、微波炉的位置。
- [ ] 聊天里说添加了微波炉，必须进入物品注册表并渲染到世界。
- [ ] Agent 回复“已添加某物”前，需要验证世界数据中确实存在该物体。

验收：
- 查询“微波炉在哪里”，员工能给出房间与坐标。
- 移动办公桌后，员工工位跟随更新。
- 柯基移动后，小文能根据坐标找过去。

## P1 恢复：Agent 执行闭环

- [ ] 聊天中出现“先做一个版本出来看看”等明确任务，应自动生成高优先级任务。
- [ ] 员工说“我立马去做”后，必须创建任务、开始执行、产出结果。
- [ ] 任务完成后在群里反馈结果，并附项目地址或入口。
- [ ] 不依赖固定关键词识别任务，要识别自然语言中的明确开发指令。
- [ ] 4 个员工按职能分工：策划、研究、开发、文案/测试。
- [ ] 任务卡需要从待办进入进行中，再到完成或阻塞。
- [ ] 阻塞任务必须说明原因，需要老板拍板时主动私聊或群聊提醒。
- [ ] 长期遗忘项目定期提醒老板是否继续、暂停或弃置。
- [ ] 聊天记录要进入长期记忆或摘要，不允许只有短期几秒上下文。
- [ ] 上下文自动压缩：保留决策、物品变更、任务状态、角色关系、老板偏好。

验收：
- 在群聊说“做一个版本看看”，看板生成任务，员工开始推进。
- 员工完成后群里给出可打开的项目链接。
- 第二天员工仍记得咖啡机、鱼缸、柯基、奶茶等世界事件。

## P2 恢复：Companyverse 与小游戏

- [ ] Companyverse 不再是空旷简陋大厅，需要从像素公司向外扩展成完整游戏宇宙。
- [ ] 从办公室出去能进入宇宙广场。
- [ ] 宇宙广场连接公司大厅、小游戏项目、生活彩蛋等。
- [ ] `Mushroom Runner` 命名统一，不再叫“水管工冒充者”。
- [ ] 小游戏支持返回，不允许进去后只能刷新。
- [ ] 小游戏支持移动端摇杆和右侧按钮。
- [ ] 帧率目标 60 FPS，至少稳定高于 50 FPS。
- [ ] 减少 DOM 过量重绘，优先 canvas 或分层渲染。
- [ ] UI 不遮挡核心内容，移动端优先。
- [ ] 内置小游戏与聊天内容、公司元素联动。

验收：
- 进入 Mushroom Runner，可用手机摇杆操作，可返回宇宙广场。
- PC 和手机帧率不低于 50 FPS。

## P2 恢复：部署与缓存一致性

- [ ] 本地、局域网、Tailscale、VPS 域名版本一致。
- [ ] `https://pix.lovenom.eu.org/` 更新后手机端能拿到新版本。
- [ ] Service Worker 或缓存要有版本号，避免旧狗、旧 UI 一直不更新。
- [ ] 增加前端版本号显示，方便确认手机是否加载新版本。
- [ ] 部署脚本同步必要文件到 VPS。
- [ ] HTTPS 证书状态正常。

验收：
- PC 和手机打开同一域名显示同一版本号。
- 更新柯基资源后，手机刷新能看到新版动画柯基。

## P2 恢复：表情包与聊天附件

- [ ] 保留像素风表情面板。
- [ ] 增加 20 款以上原创 Hermes 像素表情包。
- [ ] 表情包中文命名，比如 `[奶茶到了]`、`[柯基追尾巴]`、`[锦鲤保佑]`。
- [ ] 员工能理解表情包文本含义。
- [ ] 支持滚轮查看，手机支持滑动查看更多。
- [ ] 支持收藏表情包。
- [ ] 支持图片和文件发送。
- [ ] 柯基表情包至少包含：追尾巴、追自己影子、追鱼缸里的鱼。

验收：
- 发送 `[奶茶到了]` 时聊天中显示表情包，员工理解为奶茶到了。

## P3 优化：视觉质量

- [ ] 重新绘制员工角色，参考用户给的像素角色图。
- [ ] 员工角色区分发型、衣服、身份、性格。
- [ ] 增加阴影、光照、层次，不再像平铺方块。
- [ ] 办公室家具更真实：椅子、键盘、鼠标、主机、手机、绿植、桌面杂物。
- [ ] 员工家按性格差异化，室内家具不同。
- [ ] 员工作息、疲劳、饥饿、心情影响行为。
- [ ] 下班后员工回家，深夜睡觉或做个人活动。

验收：
- 一眼能看出每个员工是谁。
- 办公室不像空白大砖墙。

## 当前应先执行的顺序

1. P0 主世界办公室恢复稳定。
2. P0 编辑模式最终版恢复。
3. P0 手机单指拖动、双指缩放、摇杆与按钮恢复。
4. P0 聊天实时性与自动到底部恢复。
5. P1 全局返回与场景导航恢复。
6. P1 柯基状态机与碰撞恢复。
7. P1 环境感知与物品注册表。
8. P1 Agent 执行闭环。
9. P2 Companyverse 与小游戏。
10. P2 部署缓存一致性。
11. P2 表情包与附件。
12. P3 视觉质量与生活模拟。

## 每次改动前的强制检查

- [ ] 备份 `index.html`、`server.py`、关键项目页面和场景数据。
- [ ] 确认当前要改的是哪个 P 级任务。
- [ ] 只改当前任务相关文件。
- [ ] 修改后做语法检查。
- [ ] 用浏览器截图验证 PC。
- [ ] 用移动视口截图验证手机。
- [ ] 写明本次恢复了哪一项，哪些还没恢复。
