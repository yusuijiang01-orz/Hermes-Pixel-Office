# 纯UI视觉引导系统设计规范

## 设计原则

1. **零文字依赖**：所有引导信息通过视觉符号传达，不依赖任何文字说明
2. **渐进式揭示**：先展示最简单交互，复杂功能逐步解锁
3. **尊重老玩家**：引导可一键关闭，关闭后永不重复
4. **不阻塞流程**：引导动画在后台运行，不阻止玩家自由操作

## 引导元素清单

### 1. 箭头脉冲动画
- **形式**：半透明白色箭头，3帧循环（上→右→下），透明度从30%渐变到80%再回到30%
- **持续时间**：首次出现持续5秒，之后每30秒在相同位置闪烁1秒提醒
- **颜色**：默认白色，可交互对象类型对应不同色调（设施=蓝色，角色=绿色，宠物=橙色）
- **定位**：箭头尖端精确指向交互区域中心

### 2. 高亮边框
- **形式**：可交互对象外围金色脉冲边框，宽度2px
- **动画**：边框从内向外扩散的波纹效果，每1.5秒一次
- **颜色**：金色 (#FFD700)，与场景主色调明显区分
- **触发条件**：鼠标/手指悬停或触摸前自动显示

### 3. 图标语义系统
| 图标 | 含义 | 使用场景 |
|------|------|----------|
| 齿轮 | 设置/系统菜单 | 右上角常驻 |
| 手型 | 可交互对象 | 悬浮在可点击元素上方 |
| 问号 | 帮助面板 | 引导关闭后可随时查看 |
| 对话气泡 | NPC对话 | 角色头顶 |
| 向下箭头 | 滚动/查看更多 | 长列表顶部 |

### 4. 首次交互反馈
- 玩家第一次点击某个对象后，该对象的引导标记自动移除
- 移除时有轻微的"确认"动画（对象轻弹一下 + 粒子消散效果）
- 全部首次交互完成后，播放简短的庆祝动画（柯基摇尾巴）

## 技术规格

### Canvas 2D 渲染
- 引导层独立于主场景渲染，z-index最高
- 使用 requestAnimationFrame 实现平滑动画
- 性能预算：<1ms/frame（60fps下）

### 引导配置格式（JSON）
```json
{
  "scene": "office",
  "steps": [
    {
      "order": 1,
      "target": "coffee_machine",
      "type": "arrow_pulse",
      "duration": 5000,
      "icon": "hand",
      "repeat_interval": 30000,
      "once_per_session": true
    }
  ],
  "completion_reward": "corgi_tail_wag"
}
```

### 移动端适配
- 箭头最小触控区域：48x48px
- 高亮边框在触摸设备上提前0.5秒显示（预判玩家视线）
- 引导开关放置在设置菜单首屏

## 与现有系统的兼容性

### 编辑模式
- 编辑模式下引导层自动暂停（编辑本身就是一种引导）
- 保存按钮不放在鱼缸头顶（历史教训）

### 聊天系统
- 聊天界面的箭头提示与消息气泡不重叠
- 首次打开聊天时，手型图标指向输入框

## 验收标准

1. 关闭所有文字后，新玩家30秒内能找到第一个可交互对象
2. 引导动画在低端设备上保持30fps以上
3. 编辑模式拖拽与引导高亮不产生视觉冲突
4. 移动端触摸区域≥48px，无误触
